# OA 原型协作说明

## 基本原则

- `docs/OA_PRD.md` 是功能范围、页面内容和展示口径的主要依据。
- 网站原型中的页面、字段、按钮、状态和流程展示，应尽量与 PRD 保持一致。

## 双向同步规则

- 当调整 `docs/OA_PRD.md` 时，必须同步检查网站原型页面是否需要调整。
- 检查范围至少包括：页面名称、页面入口、字段、按钮、状态、提示文案、列表展示和详情展示。
- 如果 PRD 已变更而原型未更新，应把原型同步改到一致。

- 当调整网站原型页面时，必须同步检查 `docs/OA_PRD.md` 是否也需要调整。
- 如果原型里的页面结构、字段口径、按钮动作、状态定义或展示规则发生了变化，应判断这些变化是否属于产品要求变更。
- 只要这些变化会影响产品说明、验收口径或页面定义，就要同步更新 PRD，避免文档和原型脱节。

## 执行要求

- 每次开始执行新的调整前，必须先查看根目录的 `纠错记录.md`，确认不要重复犯已经被用户纠正过的错误。
- 在后续对话中，只要用户纠正了之前的错误，都必须同步记录到根目录的 `纠错记录.md` 中。
- 每次用户提出的调整需求，以及本次实际完成的改动内容，都必须同步记录到根目录的 `修改记录.md` 中。
- 不允许只改 PRD 不看原型，也不允许只改原型不回看 PRD。
- 每次改动完成前，都应做一次 PRD 与原型的一致性检查。
- 发现不一致时，优先补齐差异，并在交付说明中明确这次同步了哪些内容。
