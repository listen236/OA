# 纠错记录

## 使用规则

- 每次开始执行新的调整前，先阅读本文件。
- 用户在对话中纠正过的错误，要持续追加到本文件。
- 后续执行时，默认把这里的内容当作必须避开的已知错误。

## 记录

### 2026-06-30

- 已纠正：在 Codex 项目里，给代理看的指导文件应使用 `AGENTS.md`，不应只写在 `README.md` 里。
- 后续要求：当用户纠正了错误后，必须把该纠正同步记录到本文件，并在后续调整前先回看本文件。
- 已纠正：组织管理页不应保留顶部启用/停用标签栏；状态应放到查询区，且默认值为“启用”。
- 已纠正：组织管理页不应在页面操作区放“调整排序”按钮，排序应通过排序字段本身体现。
- 已纠正：组织管理页的操作列不应过长，行内操作展示需要尽量收短。
- 已纠正：当原型页面已调整时，不能只改页面不检查 `docs/OA_PRD.md`；必须同步回看并补齐文档口径。
- 已纠正：组织管理页的状态筛选默认值不能只显示在查询区；当默认值为“启用”时，列表也必须同步按“启用”结果过滤。
- 已纠正：员工信息详情页编辑态不能把所有字段都做成普通文本框；必须按 PRD 匹配字段类型、必填、可编辑性和选择项。
- 已纠正：梳理员工合同信息联动时，不应建议在员工信息里新增合同字段；员工信息合同字段应保持现状，只调整已有合同字段与合同与协议表的同步关系。
- 已纠正：合同与协议页面的开始日期、结束日期、签订日期应改名为合同开始日期、合同结束日期、合同签订日期，并删除试用期字段。
- 已纠正：员工详情页头像上传入口应放在顶部头像处，鼠标悬停头像即可上传；基础信息中不应再显示“员工头像”字段。
- 已纠正：员工详情页“年龄”字段应紧跟在“出生日期”后面展示。
- 已纠正：员工详情页“民族”下拉选择项不能只放少量示例，应提供完整民族选项。
- 已纠正：员工详情页中的“离职日期”和“离职原因”由离职流程驱动，不应在详情页基础信息编辑态直接编辑。
- 已纠正：新增员工页面应按员工详情字段分组展示，但不应展示工号、员工状态、司龄、工龄、离职日期、离职原因、年龄等系统生成或流程结果字段；教育经历、工作经历、家庭成员、附件存档需要在新增员工时以可添加表单块体现。
- 已纠正：新增员工抽屉中“员工主档”应命名为“基本信息”；证件照正反面上传控件不应占用过大区域；“是否试用期”不展示；教育经历、工作经历、家庭成员、附件存档标题样式应与其他分组一致；表单块删除按钮不能与输入框重叠；底部按钮应为“取消、保存”。
- 已纠正：员工详情页的附件存档也应按表单块形式展示；新增员工表单块删除按钮应与员工详情页一致使用垃圾箱图标；新增员工附件上传样式应与员工详情页一致。
- 已纠正：新增员工中的证件照正面、证件照反面也应使用与详情页一致的小上传按钮样式，不应继续使用拖拽上传框。

### 2026-07-01

- 已纠正：PRD 导出的 html 左侧目录不能只显示较浅层级；模块需求下应补充更清晰的二级菜单，如人力资源模块中的组织管理、员工管理等入口。
- 已纠正：PRD 导出的 html 不能把流程图只当作普通代码输出；`mermaid` 流程图需要在 html 中直接渲染显示。
- 已纠正：员工信息新增页和详情页的合同公司不能使用随意写死的公司列表，应取组织管理中组织类型为“公司”且启用的数据；员工信息中的“是否试用期”和“转正日期”不能在新增页或详情页手工编辑。
- 已纠正：员工档案页的新增按钮不能只弹成功提示，点击后必须出现可填写的新增交互，并能把保存后的记录展示到当前页签列表中。
- 已纠正：员工信息中的“是否试用期”字段应删除；员工状态需包含“试用期”，并按试用期结束日期、转正日期自动判断。
- 已纠正：各表单中的员工选择器不能只显示姓名，应统一展示“姓名(工号) 部门名称”；各表单中的部门选择器不能继续使用普通平铺下拉，应改为树状选择。
- 已纠正：员工信息新增页和详情页中的试用期结束日期不能只允许自动计算，必须支持自动带出后再手工修改。
- 已纠正：员工选择器中的“姓名(工号) 部门名称”只应出现在下拉选项里，选中后表单内仍应只显示姓名。
- 已纠正：员工详情中的工龄、司龄应改为保留 1 位小数并四舍五入，不再使用整数或“几年几个月”的展示方式。
- 已纠正：合同与协议表单不应继续让用户填写提醒天数；合同到期提醒应固定按提前 30 天执行。
- 已纠正：HTML 文件不需要每次修改后都重新生成，只有用户明确要求生成时才执行导出。

### 2026-07-02

- 已纠正：新增员工页面中的工号字段需要展示；工号由系统自动编号并且不可编辑，不能再按“新增员工不展示工号”的旧口径执行。
