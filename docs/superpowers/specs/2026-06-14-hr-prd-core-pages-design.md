# 核心人资页面 PRD 落地设计

## 背景

项目当前是 React 18 + TypeScript + Vite + Ant Design 5 的人力资源模块原型。现有实现以 `PageConfig` 驱动通用列表页，已经具备菜单、多标签页、查询区、工具栏、表格、字段设置、抽屉表单和详情抽屉。

新的 `docs/智能办公系统-人力资源模块PRD.md` 对核心人资模块给出了更完整的页面字段、状态、按钮、表单和业务流转要求。本设计用于把 PRD 中的核心模块落到当前系统页面中。

## 范围

本次实施覆盖核心模块：

- 组织架构：组织管理、组织架构图、岗位管理
- 员工管理：员工信息、员工档案、合同与协议
- 招聘管理：招聘需求、简历筛选、人才库、面试管理、Offer 管理
- 入转调离：入职管理、转正管理、人员调动、离职管理
- 考勤管理：补卡、请假、加班、外出、出差、考勤组、班次、排班、考勤规则、假期、打卡记录、日考勤统计、月考勤统计
- 绩效管理：考核计划、考核进度、考核模板、考核指标
- 薪酬管理：薪资审批、工资条

本次不接真实后端，不实现真实权限引擎、审批流引擎、导入导出文件处理或考勤薪资计算算法。所有状态变化在前端 mock 数据内模拟。

## 总体方案

采用“配置模型增强 + 少量专属页面”的方案。

大部分核心页面继续由通用列表框架渲染。通用框架增强后支持：

- 查询字段来自 PRD 配置。
- 列表字段分为必留字段、默认字段、可选字段。
- 字段设置可勾选 PRD 中要求但默认不展示的字段。
- 页面按钮、行内按钮、表单按钮来自 PRD 配置。
- 状态类页面支持内容区顶部状态 Tabs，并显示数量。
- 业务按钮点击后模拟状态变化，并刷新列表和 Tab 数量。
- 表单按 PRD 完整字段分区展示。

以下页面使用或保留专属组件：

- 人资工作台：卡片式 Dashboard。
- 员工详情：参考用户提供图片，使用顶部员工摘要 + 横向 Tabs + 两列详情信息。
- 组织架构图：树或组织图展示。
- 排班管理：日历/周期排班视图。
- 统计和看板类页面：指标卡、图表、明细组合布局。

## 页面状态 Tabs

状态类页面在内容区顶部展示 Tabs，格式为“状态名 数量”。数量由当前页面 mock 数据统计得到。点击状态 Tab 后筛选列表数据。

招聘管理：

- 招聘需求：草稿、已发布、招聘中、已关闭、已取消
- 简历筛选：待筛选、已通过、已淘汰、已转人才库、已发起面试
- 人才库：可联系、已邀约、已入职、暂不合适
- 面试管理：待安排、待面试、已完成、已取消
- Offer 管理：草稿、已发放、已确认、已拒绝、已撤回、已转入职

入转调离：

- 入职管理：待入职、已入职、已放弃
- 转正管理：待转正、审批中、已转正、已取消
- 人员调动：待调动、审批中、待生效、已调动、已放弃
- 离职管理：待离职、审批中、待确认、待生效、已离职、已放弃、已撤销

考勤管理：

- 补卡、请假、加班、外出、出差：草稿、审批中、已通过、已撤回
- 考勤组、班次、规则、假期：启用、停用
- 打卡记录：正常、迟到、早退、缺卡、外勤、异常
- 日考勤统计：待计算、已计算、已更新、计算异常、已锁定
- 月考勤统计：未生成、已生成、已确认、已锁定

绩效管理：

- 考核计划：草稿、已发布、评分中、审核中、已确认、已关闭、已终止
- 考核进度：待评分、评分中、待审核、已完成、已终止
- 考核模板、考核指标：启用、停用

薪酬管理：

- 薪资审批：草稿、审批中、已通过、已撤回、已归档
- 工资条：待发放、已发放、已查看、已撤回

## 配置模型

扩展当前 `PageConfig`，核心结构如下：

```ts
interface PageConfig {
  pageType?: 'crud' | 'approval' | 'report' | 'dashboard' | 'readonly' | 'special';
  searchFields: FieldConfig[];
  table: {
    columns: ColumnConfig[];
    requiredVisible: string[];
    defaultVisible: string[];
    optional: ColumnConfig[];
  };
  statusTabs?: StatusTabConfig[];
  pageActions: ActionConfig[];
  rowActions: ActionConfig[];
  form: {
    mode: 'drawer' | 'detail-page' | 'modal';
    sections: FormSectionConfig[];
    footerActions: ActionConfig[];
  };
}

interface StatusTabConfig {
  key: string;
  label: string;
  statusValues: string[];
}

interface ActionConfig {
  key: string;
  label: string;
  kind?: 'primary' | 'default' | 'danger' | 'link';
  icon?: string;
  confirm?: string;
  requiresSelection?: boolean;
  visibleWhen?: ActionCondition;
  enabledWhen?: ActionCondition;
  effect?: ActionEffect;
}
```

字段展示规则：

- `requiredVisible` 里的字段必须展示，字段设置中不可取消。
- `defaultVisible` 默认展示。
- `optional` 是 PRD 要求但默认不展示的字段，可在字段设置中勾选。
- 表格横向滚动保留，但默认视图控制在可读宽度内。

## 按钮和状态模拟

页面按钮和行内按钮都由 `ActionConfig` 渲染。点击动作时根据 `effect` 更新当前页面数据。

示例：

- 发布：招聘需求由草稿变为已发布或招聘中。
- 关闭需求：招聘需求变为已关闭。
- 转入人才库：简历状态变为已转人才库。
- 确认 Offer：Offer 状态变为已确认。
- 确认入职：入职记录由待入职变为已入职。
- 放弃入职：入职记录由待入职变为已放弃。
- 发起审批：转正、调动、离职、薪资等状态变为审批中。
- 手动办理：生命周期记录变为最终成功状态。
- 确认、锁定：考勤月报状态变为已确认或已锁定。
- 发放：工资条状态变为已发放。
- 撤回：审批中或已发放记录回退到 PRD 允许的状态。

所有模拟动作必须同步：

- 当前行状态。
- 当前列表过滤结果。
- 状态 Tab 数量。
- 状态 Tag 展示。
- 成功或失败 message。

## 表单设计

所有核心页面新增和编辑表单按 PRD 字段完整展示。字段按业务分区，不省略 PRD 中要求的字段。

普通页面默认使用 Drawer：

- 基础信息
- 业务信息
- 附件信息
- 审批信息或规则信息
- 备注/操作说明

表单底部按钮来自配置，例如：

- 保存
- 提交
- 提交审批
- 确认入职
- 手动办理
- 保存并发布
- 取消

员工信息页使用专属详情/编辑布局，参考用户提供图片：

- 顶部员工摘要：头像、姓名、员工状态、标签、部门、岗位、入职日期、司龄、右侧操作按钮。
- 横向 Tabs：基础信息、教育经历、工作经历、家庭成员、员工证书、公司内兼任、附件、直属领导、成长记录、个税申报。
- 基础信息区：两列字段展示，支持编辑入口。
- 一对多信息：在对应 Tab 内以子表格维护。
- 编辑态沿用同样分组，不把所有字段压进单个长表单。

## Mock 数据

扩展 mock 数据生成逻辑，使核心页面具备 PRD 字段和状态。

要求：

- 每个状态 Tab 至少有若干条数据，便于验证数量和状态切换。
- 业务编号、工号、部门、岗位、负责人、日期、状态等字段有稳定 mock 值。
- 金额、人数、时长等数字字段可排序和右对齐展示。
- 敏感字段可以先展示脱敏样式或普通 mock 值，不实现真实权限。

## 特殊页面处理

人资工作台：

- 保持卡片式页面。
- 按 PRD 校准待办、预警、指标、快捷入口。

组织架构图：

- 保留组织树或架构图。
- 操作包含展开、收起、缩放、下载架构图。

排班管理：

- 不使用普通表格作为主视图。
- 使用周期、班次、员工维度的排班原型视图。
- 页面操作包含班次外观、排班周期、排班设置、预置休息日、Excel 批量排班、导出记录、班次人数统计。

看板和统计页：

- 使用查询区 + 指标卡 + 图表区域 + 明细区。
- 钻取按钮按 PRD 提供，不做真实跨页深链逻辑时给出提示。

## 风险和取舍

- PRD 字段非常多，默认全展示会导致表格过宽，因此采用核心字段默认展示、其余字段进入字段设置。
- 当前是前端原型，不实现真实后端约束；所有权限、审批、导入导出、状态流转均为前端模拟。
- 为了保持可维护性，不把员工详情、排班、看板强行塞入通用列表组件。
- 当前项目不是 git 仓库，设计文档无法提交 commit，只能写入项目目录供审阅。
